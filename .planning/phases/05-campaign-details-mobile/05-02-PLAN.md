Dialog template (line 370):
```html
<Dialog
  v-model:visible="props.openWizard"
  modal
  :style="{ width: '95rem' }"
  class="wizard-dialog"
  style="max-height: 90vh; overflow-y: auto;"
```

Key issues on mobile:
1. `:style="{ width: '95rem' }"` — inline style, but `breakpoints` prop overrides width on small screens
2. `style="max-height: 90vh"` — inline style directly on the component root; limits height to 90vh, preventing 100dvh
3. PrimeVue Dialog renders these DOM elements:
   - `.p-dialog-mask` — the backdrop overlay (full-screen semi-transparent)
   - `.p-dialog` — the dialog container (has margin, border-radius, width constraints)
   - `.p-dialog-header` — sticky header area
   - `.p-dialog-content` — scrollable content area

Established CSS approach from Phase 3 (PrimeVue shadow override):
```css
/* From 03-foundation */
::v-deep(.p-dialog) {
  box-shadow: none !important;
  border: 1px solid #e5e7eb;
}
```
Use the same `!important` strategy where PrimeVue applies inline styles.

The `.wizard-dialog` class is applied to the Dialog component — use this to scope all deep overrides to only this dialog instance.

In the `<style scoped>` block, add a new `@media (max-width: 768px)` block at the END of the style section (after the dark mode block), before the closing `</style>` tag.

```css
/* ===========================
   Mobile: Full-screen wizard
   =========================== */
@media (max-width: 768px) {
  /* Hide the backdrop mask — dialog IS the full screen */
  ::v-deep(.p-dialog-mask) {
    background: transparent !important;
    backdrop-filter: none !important;
  }

  /* Make the dialog fill the full viewport */
  .wizard-dialog ::v-deep(.p-dialog) {
    width: 100vw !important;
    height: 100dvh !important;
    max-width: 100vw !important;
    max-height: 100dvh !important;
    margin: 0 !important;
    border-radius: 0 !important;
    top: 0 !important;
    left: 0 !important;
    transform: none !important;
    position: fixed !important;
    display: flex !important;
    flex-direction: column !important;
  }

  /* Header stays at top, does not scroll */
  .wizard-dialog ::v-deep(.p-dialog-header) {
    flex-shrink: 0;
    padding: 1rem 1rem 0.75rem;
    border-radius: 0;
  }

  /* Content area fills remaining height and scrolls */
  .wizard-dialog ::v-deep(.p-dialog-content) {
    flex: 1 1 auto;
    overflow-y: auto;
    padding: 0.75rem 1rem;
    max-height: none !important;
  }

  /* Tighten up wizard content padding on mobile */
  .wizard-content {
    padding: 0;
  }

  /* Reduce heading size on mobile */
  .wizard-dialog ::v-deep(.p-dialog-header) h2 {
    font-size: 1.5rem;
    margin-bottom: 0.75rem;
  }
}
```

**Critical notes:**

1. **The `max-height: 90vh` inline style** on the Dialog component (the `style="max-height: 90vh; overflow-y: auto;"` attribute on line 372) is an inline style on the Vue component root, NOT on `.p-dialog` itself. PrimeVue typically maps this to the `.p-dialog` element. Use `max-height: 100dvh !important` in the CSS override to win over it. If it persists in the browser after testing, the fallback is to conditionally bind the style:
   ```html
   :style="{ 'max-height': '90vh', 'overflow-y': 'auto' }"
   ```
   For now, attempt the CSS-only approach — update the template attribute only if the CSS override fails during verification.

2. **`100dvh` vs `100vh`** — use `dvh` (dynamic viewport height) to account for mobile browser chrome (address bar). Safari on iOS collapses/expands the address bar, and `dvh` tracks the actual available height. This is the correct unit for mobile full-screen.

3. **Scoping** — scope `.p-dialog` overrides under `.wizard-dialog ::v-deep(...)` to avoid affecting any other PrimeVue dialogs that may exist in the app (the Popover/chamber list in Step 2 is a `<Popover>` not a `<Dialog>`, so no collision). Use bare `::v-deep(.p-dialog-mask)` for the mask since the mask is a sibling of the dialog, not a child.

4. Do NOT modify the Dialog template attributes (`:style`, `:breakpoints`) unless the CSS-only approach fails during verification.
