<task type="auto">
  <name>Task 2: Fix WIZD-02 + WIZD-01 — mobile footer flex-shrink and font preconnect</name>
  <files>src/dialogs/wizard/WizardView.vue, index.html</files>
  <action>
Two changes — one CSS, one HTML:

**Change A — Mobile footer override in WizardView.vue (WIZD-02):**
Inside the `@media (max-width: 768px)` block, add a footer rule immediately before the closing `}` of the media block (which is followed by `</style>`). Place it after the `/* Tighten up wizard content padding on mobile */` section:

```css
  /* Footer stays at bottom of viewport — does not compress scrollable content area */
  .wizard-dialog ::v-deep(.p-dialog-footer) {
    flex-shrink: 0;
  }
```

This mirrors the pattern already used for `.p-dialog-header` (`flex-shrink: 0`) in the same block. The dialog is `flex-direction: column` on mobile — without this, the footer's default flex behavior is `flex-shrink: 1`, which can compress the content area.

**Change B — Add fonts.gstatic.com preconnect in index.html (WIZD-01 perf):**
After the existing `<link rel="preconnect" href="https://fonts.googleapis.com">` on line 7, add:

```html
<link rel="preconnect" href="https://fonts.gstatic.com" crossorigin>
```

Google Fonts requires both preconnect hints — `fonts.googleapis.com` for the CSS stylesheet request, and `fonts.gstatic.com` (with `crossorigin` attribute) for the actual font binary files. Without this, the font file TCP/TLS handshake happens after the CSS is parsed, increasing the FOUT window for the Outfit typeface.
  </action>

    <automated>grep -n "p-dialog-footer\|fonts.gstatic" src/dialogs/wizard/WizardView.vue index.html</automated>
  </verify>

    - `fonts.gstatic.com` appears in index.html as a preconnect link with `crossorigin` attribute
    - `npm run build` passes without errors
  </done>
