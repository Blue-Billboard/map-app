## Unsatisfied Requirements

### WIZD-05 — Zero box shadows on any wizard element

**Status:** Unsatisfied (live violation found by integration checker)

**Evidence:**
The Chamber Member discount info `<Popover ref="op" class="surface-card shadow-2 border-round">` at WizardView.vue line 589 renders with Aura theme's default popover shadow injected via CSS variable `{overlay.popover.shadow}` resolving to `0 4px 6px -1px rgba(0,0,0,0.1), 0 2px 4px -2px rgba(0,0,0,0.1)`.

- Phase 3's `box-shadow: none !important` override targets `:deep(.p-dialog)` — it does not cover `:deep(.p-popover)`
- `shadow-2` class is a dead PrimeFlex reference (PrimeFlex not in package.json), so it does nothing
- Aura's shadow token on `.p-popover` is unaffected by any scoped style in WizardView.vue
- The popover renders with a real drop shadow whenever a user clicks the Chamber Member info icon in Step 2

**Fix:** Add `:deep(.p-popover) { box-shadow: none !important; border: 1px solid #e5e7eb; }` to WizardView.vue scoped styles.

## Partial Requirements

### WIZD-02 — Mobile full-screen wizard

**Status:** Partial

**Evidence:**
Mobile media query at line 1718 covers `.p-dialog`, `.p-dialog-header`, and `.p-dialog-content` with correct `100vw × 100dvh` full-screen overrides. However, `.p-dialog-footer` (containing the Close button) has no explicit mobile override. In Aura's dialog structure, the footer is a flex child of `.p-dialog` — it likely renders at the bottom correctly, but its height is not factored into the `flex: 1 1 auto` content scroll calculation. If the footer has padding or minimum height, the content area could be shorter than expected.

**Fix (optional):** Add `::v-deep(.p-dialog-footer) { flex-shrink: 0; }` inside the mobile media block, or hide the footer on mobile if "Close" is redundant with the mobile back navigation.
