# Requirements: MapApp — Blue Billboard Venue Map

**Defined:** 2026-03-03
**Core Value:** The map must feel as premium as the venues it displays — first impressions matter.

## v1.1 Requirements

Quote wizard redesigned with a bold flat design system — no shadows, no gradients on components, Outfit font, full-bleed images, and a full-screen mobile experience.

### Wizard Wrapper & Navigation

- [x] **WIZD-01**: User sees Outfit font applied throughout the entire wizard (all steps, all text)
- [x] **WIZD-02**: User on mobile sees the wizard fill the full viewport (100vw × 100dvh, no dialog chrome, no border-radius, no margin)
- [x] **WIZD-03**: User sees a custom flat step indicator replacing PrimeVue Steps (numbered blocks: active=navy, complete=navy+checkmark, pending=gray)
- [x] **WIZD-04**: User sees a snappy step transition animation when navigating between steps (opacity + translateX slide, 200ms)
- [x] **WIZD-05**: User sees zero box shadows on any wizard element (buttons, cards, inputs, containers)

### Venue Selection (Step 1)

- [x] **VENUE-01**: User sees venue cards with full-bleed hero images (image fills card entirely, venue name and city overlaid on a flat navy color band at the bottom)
- [x] **VENUE-02**: User sees selected venue cards with a flat selected state (solid navy border + bg-blue-50 tint, no shadow)
- [x] **VENUE-03**: User sees a flat search/filter bar (gray-100 background, no border, no shadow)
- [x] **VENUE-04**: User sees selected venue count as a bold navy color chip with white text (not a soft pill)

### Campaign Details (Step 2)

- [x] **CAMP-01**: User sees discount cards with flat styling (no shadow, no gradient — hover intensifies background color + scale-[1.02])
- [x] **CAMP-02**: User sees the duration selector using flat input styling (gray-100 background, no floating border)
- [x] **CAMP-03**: User sees selected venues displayed as flat chips (no shadow)

### Quote & Pricing (Step 3)

- [x] **QUOT-01**: User sees the featured tier card (Most Popular) as a solid navy background block with white text — poster-style
- [x] **QUOT-02**: User sees pricing cards using flat color block style (no box shadows, no gradient borders, solid backgrounds)
- [x] **QUOT-03**: User sees price amounts displayed extra-large (text-5xl, font-bold, tight leading)
- [x] **QUOT-04**: User sees the CTA section as a solid gray-900 block with white text
- [x] **QUOT-05**: User sees bold geometric background decoration on the pricing step (large low-opacity circles, absolutely positioned)
