# Requirements: MapApp — Premium Map Redesign

**Defined:** 2026-03-01
**Core Value:** The map must feel as premium as the venues it displays — first impressions matter.

## v1 Requirements

### Map Tiles

- [x] **TILES-01**: Light mode displays Stadia Maps Alidade Smooth tile layer (soft pastel, minimal cartography)
- [x] **TILES-02**: Dark mode displays Stadia Maps Alidade Smooth Dark tile layer (proper dark base, not inverted)
- [x] **TILES-03**: CSS invert filter hack removed from dark mode map tile implementation
- [x] **TILES-04**: Tile layer switches correctly when system color scheme changes

### Venue Pins

- [x] **PINS-01**: All venue pins use a single unified design (no Blue/Gold/Platinum visual distinction)
- [x] **PINS-02**: Existing billboard icon is preserved within the new pin design
- [x] **PINS-03**: Pin design feels premium — uses shadow, glow, or refined wrapper styling
- [x] **PINS-04**: Multiple pin color/style variants are presented for user approval before finalizing
- [x] **PINS-05**: Pin hover state gives clear visual feedback (scale, brightness, or shadow change)
- [x] **PINS-06**: Pins render cleanly in both light and dark map modes

### Code Cleanup

- [x] **CODE-01**: Level color bindings removed from pin rendering logic (levelColour() no longer drives pin appearance)
- [x] **CODE-02**: useVenueList composable levelColour() function retained (still used internally) but decoupled from map markers

## v2 Requirements

### Enhancements

- **ENH-01**: Pin clustering for dense venue areas
- **ENH-02**: Animated pin entrance on map load
- **ENH-03**: Custom map attribution styling to match brand

## Out of Scope

| Feature | Reason |
|---------|--------|
| Switching map library (e.g., MapLibre, Google Maps) | Leaflet works, visual upgrade only |
| Adding new map interactions | Separate milestone |
| Quote wizard changes | Not related to visual redesign |
| Custom Stadia map style editor | Free tier styles are sufficient |
