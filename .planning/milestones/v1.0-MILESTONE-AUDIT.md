| Phase 2: `mouseover`/`mouseout` events | Phase 2: `.pin-inner scale(1.12)` CSS | Leaflet event → `pin-hovered` class toggle | ✓ Wired |

### Dark Mode Coordination

Phase 1 (tile swap) and Phase 2 (pin color) respond to OS dark mode through independent but complementary mechanisms:
- Phase 1: JavaScript `window.matchMedia` change listener imperatively swaps tile layers
- Phase 2: CSS `@media (prefers-color-scheme: dark)` passively overrides pin color

These mechanisms cannot get out of sync — the CSS path has zero latency (no JS call stack). The architecture is correct and intentional.

## Tech Debt

### Phase 2: Premium Pins

**Item 1: transform-origin discrepancy**
- CSS: `transform-origin: center center` (pin scales from visual midpoint)
- Plan intent: `transform-origin: center bottom` (pin grows upward from tip, tip stays pinned to map coordinate)
- Impact: During hover, the pin tip shifts slightly downward (~4px at scale 1.12). Cosmetic only. User confirmed hover is "perfect" during execution.
- Action: Optional correction for v1.1 if geographic-anchor hover is desired.

**Item 2: levelColour dead import**
- `levelColour` is destructured from `useVenueList()` in App.vue but never called.
- Intentional per CODE-02 — retained for future consumers, annotated with `// retained:` comment.
- Impact: Zero at runtime. Tree-shaking eliminates it from bundle. May generate a linter warning if unused-variable rules are enforced.
- Action: None required unless strict linting is added.
